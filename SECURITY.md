# Security Policy

This document outlines the security policy for the `MaguttiBuilders` project.

## Reporting a Vulnerability

Security issues should not be reported through public GitHub issues.

If you discover a security vulnerability in `MaguttiBuilders`, please report it privately by emailing the maintainer at [hello@magutti.com](mailto:hello@magutti.com).

When reporting a vulnerability, please include as much information as possible:

* a clear description of the issue
* steps to reproduce the problem
* the affected version or commit
* the Laravel and PHP versions involved
* any relevant logs, screenshots, or proof of concept

I will review the report and try to respond with the next steps as soon as possible.

## Supported Versions

Security updates are provided for the latest maintained version of this project.

| Version        | Supported |
| -------------- | --------- |
| Latest version | Yes       |
| Older versions | No        |

## Scope

This security policy applies only to the code maintained in this repository.

Security issues related to Laravel, PHP, Composer packages, or other third-party dependencies should be reported to the maintainers of those projects.

## Disclosure Policy

When a security vulnerability is reported, the maintainer will:

* confirm the issue and determine the affected versions
* review the code for similar problems
* prepare a fix when applicable
* publish a new release or patch as soon as reasonably possible

Please do not publicly disclose the vulnerability before a fix or mitigation has been made available.

## Comments on this Policy

Suggestions to improve this policy are welcome through a pull request.
